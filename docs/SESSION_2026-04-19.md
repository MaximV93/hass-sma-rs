## MIS multi-inverter (0.1.46) — biggest single feature of the session

Before today: hass-sma-rs polled ONE inverter per `inverters:` entry via
its own RFCOMM socket. Maxim's real installation is one BT repeater
(MAC `00:80:25:21:32:35`) fronting two inverters — the old config would
have needed two direct BT connections to unique MACs, which don't exist.

After today: optional `devices:` sub-list per `inverters:` entry
enumerates the inverters reachable through one RFCOMM socket using
per-request app_serial addressing (SBFspot's MIS pattern).

Live results (09:55 local, first poll cycle after deploy):
- zolder (SB 3000HF-30, serial 2120121246): 139 W AC, all 11 queries retcode=0
- garage (SB 2000HF-30, serial 2120121383): 1071 W AC, all 11 queries retcode=0
- 22 queries/tick × 60 s cycle, ~8 s per cycle (52 s headroom)
- 58 HA sensors live (29 per device), 4 HA alerts armed

This gives Maxim full rooftop visibility for the first time since
haos-sbfspot was stopped on 2026-04-18. Garage was completely
invisible to hass-sma-rs until now.

## Topology discovery

Legacy haos-sbfspot logs reveal the real shape:

```
hci0 → BT Repeater @ 00:80:25:21:32:35 (serial 170004597)
         ├── SB 3000HF-30 "zolder"  serial 2120121246
         └── SB 2000HF-30 "garage"  serial 2120121383
```

Topology log from 0.1.44 shows 3 SMA-OUI MACs + host. Live probing:

- `00:80:25:21:87:CA` → reports zolder (repeater relay path A)
- `00:80:25:21:32:35` → reports zolder (configured — repeater relay path B)
- `00:80:25:1E:11:27` → EHOSTDOWN in solo probe (most likely garage's own BT, asleep-or-power-save)

**Garage is BT-reachable** — serial 2120121383 showed up in logon-broadcast
replies during the probe attempts (retcode 0x0001 "session already active"
after we left zombie sessions, but that still proves the device is
online and accepting the broadcast).

Earlier docs (including my first DISCOVER.md draft) called the second
inverter "zonneveld" — that was a misremembering. It's "garage". Fixed.

## Known limitations

- **Event log query deferred** (ADR 0004). Multi-packet reply handling
  + ctrl=0xE0 + full TagList table. Not shipped, not started.
- **Archive sink not wired for this install.** TimescaleDB + CSV sinks
  exist (0.1.40) but user hasn't enabled them. MQTT + HA LTS covers
  the basics.
- **No per-device password override for MIS.** `DeviceCfg.password`
  is defined in the schema but not yet consumed — currently the
  parent `InverterCfg.password` is used for all devices. Works for
  Maxim's setup; extend if someone has mixed-password installs.
